1. タイムゾーン設定

Laravelのデフォルトタイムゾーンは UTC のため、日本時間より9時間ズレて保存される。

修正箇所

config/app.php

'timezone' => 'Asia/Tokyo',

設定キャッシュクリア

sail artisan config:clear
DB (MySQLコンテナ) 側のタイムゾーンは UTC のままでOK。Laravel が読み書きの際に変換してくれる。

2. 詳細画面の実装

コントローラー (show)

public function show(Todo $todo)
{
    return view('todos.show', compact('todo'));
}

ルートモデルバインディング: 引数に Todo $todo と書くだけで、URLの {todo} (ID) から自動的にレコードを取得。存在しなければ404。

一覧画面のタイトルをリンク化

<td class="px-6 py-4">
    <a href="{{ route('todos.show', $todo) }}" class="text-blue-600 hover:underline">
        {{ $todo->title }}
    </a>
</td>

最終的な詳細画面のレイアウト

┌─────────────────────────────────────────────┐ │ ← 一覧に戻る │ ├─────────────────────────────────────────────┤ │ 作成 2026/04/14 05:12 更新 05:12 ●未着手 │ ← ヘッダー ├─────────────────────────────────────────────┤ │ │ │ 筋トレをする 開始日 完了日 │ ← タイトル + 日付を同列 │ 2026/04/14 2026/04/14 │ │ │ │ 詳細 │ │ 今日は下半身 │ ← 左寄せ │ │ ├─────────────────────────────────────────────┤ │ [ 編集 ] │ └─────────────────────────────────────────────┘

show.blade.php のポイント

  • カラーは gray ではなく slate を使用（青みがかった今っぽい色）
  • ステータスバッジ: 丸いドット + ring-1 で縁取り
  • 角丸は rounded-2xl / rounded-xl でやわらかく
  • 欠損値は —（全角ダッシュ）
  • 日付は tabular-nums で幅を揃える

3. 編集機能の実装

設計方針: フォームを共通パーシャルに

新規作成と編集でフォームの構造がほぼ同じなので、_form.blade.php として共通化して再利用。

resources/views/todos/ ├── index.blade.php ├── show.blade.php ├── _form.blade.php ← 共通フォーム (新規作成) ├── create.blade.php ← @include で _form を呼び出す └── edit.blade.php ← @include で _form を呼び出す

UpdateTodoRequest 作成

sail artisan make:request UpdateTodoRequest

ルールは StoreTodoRequest と同じ内容をコピー。

コントローラー (edit / update)

use App\Http\Requests\UpdateTodoRequest;

public function edit(Todo $todo)
{
    return view('todos.edit', compact('todo'));
}

public function update(UpdateTodoRequest $request, Todo $todo)
{
    $todo->update($request->validated());

    return redirect()->route('todos.show', $todo)
        ->with('success', 'ToDoを更新しました');
}

_form.blade.php（抜粋）

<form action="{{ $action }}" method="POST" class="...">
    @csrf

        @method($method)
    @endisset

    <input type="text" name="title" value="{{ old('title', $todo->title ?? '') }}" ...>
    ...
    <button type="submit">{{ $submit }}</button>
</form>

edit.blade.php

@section('title', 'ToDo編集')

@section('content')

@include の仕組み

渡す変数_form.blade.php での用途
$action<form action="..."> のURL
$cancelキャンセルボタンのリンク先
$submit送信ボタンの文字
$method編集時のみ PUT を出力

フォームでの old() の書き方

value="{{ old('title', $todo->title ?? '') }}"

第2引数のデフォルト値の仕組みで、以下のように動く:

状況表示される値
バリデーションエラーで戻された直前の入力値 (old)
編集画面の初回表示既存のtodoの値
新規作成画面空文字

本日詰まったポイント

トラブル①: 詳細のテキストが左寄せにならない

現象

詳細(description)の表示で、テキストの先頭に謎のインデントが入ってしまう。

原因

whitespace-pre-wrap は Bladeテンプレート内の空白もそのまま表示 してしまう。VSCode のフォーマッタが自動で改行・インデントを入れると、その空白が出力される。

pre-wrap:

改行・連続空白をすべて保持 → インデント空白も表示される

pre-line:

改行は保持、連続空白は1つに圧縮 → インデント影響なし

解決方法

whitespace-pre-wrap を whitespace-pre-line に変更。加えて trim() で前後の空白を除去。

<div class="text-slate-700 leading-relaxed whitespace-pre-line break-words">
    {{ trim($todo->description ?? '—') }}
</div>

white-space プロパティの違い

値改行連続空白インデント影響
pre-wrap保持保持受ける ❌
pre-line保持1つに圧縮受けない ✅
(なし)無視1つに圧縮受けない

トラブル②: 更新時に "Method Illuminate\Http\Request::validated does not exist"

現象

編集画面から更新を実行するとエラー。

原因

コントローラーの update メソッドの引数の型が Request のまま。make:controller -r で自動生成されたデフォルトの型ヒントを書き換え忘れていた。

NG: 
public function update(Request $request, Todo $todo)
{
    $todo->update($request->validated());
}

Request クラスには validated() メソッドがない

OK: 
public function update(UpdateTodoRequest $request, Todo $todo)
{
    $todo->update($request->validated());
}

FormRequest を型ヒント指定すれば validated() が使える

解決方法

  1. 引数の型を UpdateTodoRequest に変更
  2. ファイル上部に use App\Http\Requests\UpdateTodoRequest; を追加
教訓: コントローラーに自動生成された Request 型は 手動で FormRequest に差し替える のを忘れないこと。validated() を使いたい場合は必須。

動作確認

  1. 一覧画面でタイトルクリック → 詳細画面に遷移
  2. 詳細画面の「編集」ボタンクリック → 編集画面に遷移
  3. 編集画面には既存の値が入っている
  4. 値を変更して「更新」 → 詳細画面に戻り、更新された値が表示される
